1. What this is

tier1-audit is a static-analysis tool that reads one .xlsm file and produces a deterministic report — pillar cells, magic-number anomalies, VBA module classification, code smells, complexity score. It runs entirely on your machine and makes zero network calls during audit.

An optional Track B (BYOA — Bring Your Own AI) mode emits a structured prompt + dossier you feed into your own Copilot Chat or Claude session to get richer business-semantic narratives. The tool itself never calls an LLM, never holds an API key, and never proxies LLM traffic.

2. Compliance note (for IT)

If your IT or security team needs justification, the text below is copy-paste ready.

tier1-audit is a Python tool that performs static analysis of one Excel workbook on the user's local machine. Specifically:

  • It runs entirely local. There are no API keys, no telemetry, and no network calls during audit. Verifiable by running with the network disabled, or by inspecting outbound traffic with Wireshark / corporate proxy logs.
  • The runtime depends on three open-source Python libraries, each widely-deployed and permissively-licensed:
    • openpyxl (MIT) — the standard Python library for reading .xlsx/.xlsm files; ~10M downloads/month.
    • oletools (BSD) — used by SOC analysts and malware researchers for VBA extraction.
    • formulas (EUPL-1.1) — Excel formula tokenizer; we use the tokenizer only, no formula evaluation.
  • No macro is ever executed. No formula is ever evaluated. VBA source is parsed as text.
  • Zero LLM API calls in the tool itself. Track B (optional) uses the user's existing Copilot or Claude session — i.e. their own pre-approved chat tool — and is a manual paste, not a network call from this package. There is no requests, httpx, anthropic, or openai import in the source tree, and tests guard this.
  • All outputs land on local disk in a directory the user picks. A --sanitize flag redacts non-formula cell values for safe sharing of the report.
  • Source is < 2,500 lines of Python under src/tier1_audit/; auditable in an afternoon.

3. Prerequisites

  • Python 3.9 or newer (verify: python3 --version).
  • Git (only if you choose git clone; ZIP download works otherwise).
  • VS Code recommended for Track B; any terminal works for Track A.

4. Quick Start — Track A (simplest)

Track A is a deterministic static audit. No LLM. No network. This is the floor of capability and is recommended as your first run. 

# 1. Get the code (clone OR download & unzip)
git clone https://github.com/haxita/xlsm-audit.git
cd xlsm-audit

# 2. Install (creates a venv first; recommended)
python3 -m venv .venv
source .venv/bin/activate          # macOS/Linux
# .venv\Scripts\Activate.ps1       # Windows PowerShell
pip install .

# 3. Run on your file
audit-xlsm path/to/your-file.xlsm --sanitize --out-dir ./out

# 4. Open the report
open ./out/audit.html              # macOS
# start .\out\audit.html           # Windows
# xdg-open ./out/audit.html        # Linux

The audit takes anywhere from a few seconds (small workbooks) to a couple of minutes (large multi-sheet workbooks with VBA). Output files land in ./out/: audit.md, audit.html, audit.json.

5. Optional — Track B (LLM-enriched audit)

Track B uses your existing Copilot Chat or Claude session to produce business-semantic narratives ("Implements MRP netting; hardcodes 5% loss factor") instead of mechanical ones ("data-loader, 80 lines, reads sheet X"). Two paths.

5a. Auto path (recommended) — Copilot custom agent

This repo ships a custom Copilot agent at .github/agents/xlsm-audit.agent.md. With the repo open in VS Code:

  1. Open Copilot Chat (Cmd+Alt+I on macOS, Ctrl+Alt+I on Windows/Linux).
  2. Click the agent dropdown in the chat panel and select xlsm-audit. (If the dropdown is empty, run Developer: Reload Window from the Command Palette.)
  3. Type the absolute path of your .xlsm file and send.
  4. Approve terminal-command prompts as they appear (Copilot will ask before each audit-xlsm invocation).
  5. When done, the agent opens audit-output/audit-enriched.html automatically.

The agent runs the seven-step pipeline you would otherwise run by hand: audit-xlsm --harness, read the prompt and dossier, produce the JSON response, write responses.json, run audit-xlsm --ingest, open the result.

Don't have Copilot in your VS Code? Use the manual fallback below.

5b. Manual path (fallback) — the original Track B flow

# Step 1 — extract the harness
audit-xlsm path/to/file.xlsm --harness --out-dir ./audit-output

# Step 2 — open audit-output/prompt.md, paste into Copilot Chat /
#          Claude / ChatGPT, attach audit-output/dossier.json, copy
#          the JSON response, save as audit-output/responses.json

# Step 3 — splice the narratives back in
audit-xlsm path/to/file.xlsm --ingest audit-output/responses.json --out-dir ./audit-output
open ./audit-output/audit-enriched.html

The full walkthrough lives at docs/harness-guide.md in this repo.

6. Troubleshooting

If you hit a problem, please check this list before pinging me — most issues are environmental, not bugs.

Install issues

1. pip install . fails — PyPI access blocked. Inside companies with no public PyPI access, use an internal mirror: pip install --index-url https://<INTERNAL_MIRROR>/simple/ .. The three runtime deps (openpyxl, oletools, formulas) are common; if they're missing from the mirror, ask IT to add them. See docs/friend-test-setup.md Path 3.

2. Old pip rejects PEP 660 / editable install. Use pip install . (not pip install -e .) — the non-editable install bypasses the PEP 660 hook entirely.

3. requires-python mismatch. The package needs Python 3.9+. If your machine has Python 3.8 or older, install Python 3.9+ side-by-side and use it for the venv: python3.9 -m venv .venv.

4. git clone is blocked. Download the repo as a ZIP from GitHub (Code → Download ZIP), unzip, and run pip install . from the unzipped folder.

5. SSL cert verify failures (corporate MITM proxy). Set HTTPS_PROXY / HTTP_PROXY env vars or pass --proxy http://corp.proxy:port to pip. If the proxy uses a self-signed cert, ask IT to add it to the system trust store or use --trusted-host.

6. PowerShell "running scripts is disabled". On Windows: Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned, then re-activate the venv. Reversible, current-user only.

Run-time issues

7. Audit crashes with "Failed to extract VBA macros". The file is encrypted / password-protected. Either Save As without password in Excel and re-run, or install msoffcrypto-tool separately, decrypt the workbook, and run audit on the decrypted copy. Encrypted-xlsm support is not yet built in.

8. ImportError: cannot import name 'X' from 'numpy'. Pre-installed numpy mismatched with formulas. Run pip install --upgrade --force-reinstall numpy formulas inside the venv.

9. Mermaid diagrams don't render in audit.html. The default HTML references the public Mermaid CDN at view time. If your machine has no internet at view time, re-run with --mermaid-inline to embed the JS into the file.

10. Path-too-long on Windows (deep OneDrive folders). Copy the .xlsm to a shorter path (e.g. C:\Temp\file.xlsm) and audit from there.

Track B / agent issues

11. Copilot Chat doesn't show the xlsm-audit agent in the dropdown. Ensure the repo is opened in VS Code (not just an arbitrary folder) so .github/agents/xlsm-audit.agent.md is in the workspace. Then run Developer: Reload Window from the Command Palette.

12. Copilot returns prose instead of JSON. Some models drop earlier instructions on long prompts. Either: (a) re-prompt with "Output ONLY a single top-level JSON object, no markdown fences"; or (b) fall back to the manual Track B path (Section 5b) — the --ingest step tolerates partial responses and gracefully degrades unfilled markers.

13. Agent stalls mid-flow / refuses a step. Tell the agent "continue from step N". If it still stalls, switch to the manual path. The manual flow is the always-works floor.

14. Company computer blocks all third-party Python. The audit tool can be inspected and run as a single set of plain .py files (no compiled extensions beyond the standard wheel for cryptography, a transitive dep of oletools). If you can't install at all, the audit.json output of a colleague's run is still useful raw data you can review.

7. What feedback would help most

Quantitative bug reports are welcome but the most useful feedback is qualitative reactions to the report's content. In rough priority order:

  • Workflow Guide accuracy — does the auto-generated "what this workbook is for / how it's used" section match how you actually use the file? If not, where does it go wrong?
  • Pillar cells — did the inline value+label identify cells you yourself would have flagged as critical? Any obvious ones it missed?
  • Domain template hits — was capacity-planning / logistics-routing / inventory-supply-chain correctly detected? If miscategorised, what would you have called it?
  • Track A vs Track B — if you tried both, was the LLM-enrichment worth the friction? Did the auto-agent path or the manual path work better for you?
  • False alarms — any findings that are noise we should suppress? Concrete examples (sheet name + finding type) help most.
  • Blind spots — any real risks the audit missed? Worth a callout even if you can't say exactly what would catch them.
  • Install pain — which path in docs/friend-test-setup.md worked / didn't / should be easier?

8. How to send feedback

Two channels, in order of preference:

  • GitHub Issues at https://github.com/haxita/xlsm-audit/issues — pulse-checked. Best for "here's a concrete bug or noise example". Paste a snippet from a --sanitize-mode audit.md; don't paste the original workbook.
  • Direct to Michael — via whatever channel you already have. Best for qualitative reactions, half-formed thoughts, "this didn't feel right but I'm not sure why" comments.

Please don't send the original .xlsm file. We don't need it. The audit.md excerpt or audit.json is sufficient. --sanitize exists precisely for this case.

9. Thanks & expectations

This is a beta. The tool will be wrong about things; the agent flow may stall on your first try; the install may want a Path 2 instead of a Path 1. None of that is wasted — every reproducible thing you can send back becomes a fix or a doc clarification, which is the entire point of this round of friend-testing. Half-tested vague reports ("the X felt off") are better than no report.
